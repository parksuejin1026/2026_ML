# 📊 구독 해지 예측 프로젝트 (Subscription Churn Prediction)

본 문서는 머신러닝을 활용하여 사용자의 구독 해지 여부를 예측하는 초기 모델 설계 및 결과 로그입니다.

---

## 1. 문제 정의서 (Problem Definition)

### 💡 해결할 문제
현대인은 다양한 구독 서비스를 이용하지만, 실제 사용량과 관계없이 자동 결제가 이루어지면서 불필요한 지출(구독 피로감)이 발생합니다. 기업 입장에서는 이러한 고객들이 어느 순간 갑작스럽게 구독을 해지(Churn)하는 현상을 방어해야 합니다.

### 🎯 예측 목표
사용자의 이용 패턴 데이터를 분석하여 **다음 달 구독 유지 여부를 이진 분류로 예측**합니다.
- **클래스 0 (해지)**: 서비스에서 이탈할 위험이 높은 고객 (주요 선별 타겟)
- **클래스 1 (유지)**: 서비스를 계속 이용할 고객

예측 결과를 바탕으로 해지 위험군을 조기에 선별하여 마케팅 방어 전략의 기반을 마련하는 것이 최종 예측 목표입니다.

---

## 2. 데이터 설명 (Data Description)

- **출처**: 초기 학습 모델 검증을 위해 가상의 사용자 시나리오를 가정한 **AI 자체 생성 난수 데이터(`mock_data.csv`)**를 사용합니다.
- **크기**: 고객 데이터 5,000건 (해지 20%, 유지 80% 불균형 비율 적용)

### 📊 주요 변수
| 변수명 | 설명 | 비고 |
|---|---|---|
| **Monthly_Fee** | 월 구독료 | 해지군에서 다소 높은 경향 |
| **Access_Days** | 월 접속 일수 | 해지군에서 적은 경향 |
| **Usage_Time** | 월 누적 이용 시간 | 해지군에서 짧은 경향 |
| **Content_Count** | 콘텐츠 소비량 | - |
| **Customer_Inquiry** | 고객센터 문의 횟수 | 해지군에서 잦은 경향 |

### 🧠 파생 변수 (Feature Engineering)
데이터 특성을 구체화하기 위해 다음 변수들을 새롭게 파생시켰습니다.
- **Inquiry_Rate (문의율)** = Customer_Inquiry / (Access_Days + 1)
- **Value_Score (가성비 점수)** = Usage_Time / Monthly_Fee

---

## 3. 기본 모델 실행 (Baseline Model Execution)

첫 번째 ML 모델로 선형 기반 분류 모델인 **로지스틱 회귀 (Logistic Regression)** 알고리즘을 선정하여 초기 학습을 진행했습니다. 외부 CSV 데이터를 읽어들여 학습하도록 파이썬 가상환경(`venv`)과 주피터 노트북 환경(`churn_prediction.ipynb`)을 구축했습니다. 

- **데이터 전처리**: 훈련용(Train)과 테스트용(Test) 데이터를 8:2로 분리한 뒤 피처 스케일링(StandardScaler) 적용.
- **모델 세팅**: `sklearn` 라이브러리의 `LogisticRegression` 학습 파이프라인 적용.

---

## 4. 성능 지표 해석 (Metrics Interpretation)

기본 모델에 대한 테스트 세트의 평가 결과는 다음과 같이 측정되었습니다. (클래스 0인 '구독 해지' 포착에 초점을 맞춰 측정)

- **정확도 (Accuracy)**: `0.998` 
  - 전체 중에 단순하게 정답을 맞힌 비율. (단, 해지와 유지가 불균형한 데이터에서는 단독으로 신뢰하기 어려움)
- **정밀도 (Precision)**: `0.995` 
  - 모델이 '해지'할 것으로 예측한 고객 중 실제로 '해지'한 고객의 비율. (무분별한 마케팅 비용 낭비를 막는 지표)
- **재현율 (Recall)**: `0.995` 
  - 실제 '해지' 대상 고객 중에서 모델이 '해지'한다고 찾아낸 비율. **(고객 이탈을 방어하기 위해 비즈니스적으로 가장 중요한 지표)**
- **F1-점수 (F1-Score)**: `0.995`
  - 정밀도와 재현율의 조화 평균으로, 모델이 치우치지 않고 균형 잡힌 분류를 하는지에 대한 증명.
- **ROC-AUC 점수**: `0.999`
  - 전반적인 이진 분류 능력.

> *참고*: 현재 모델의 성능이 비정상적으로 높게 산출된 이유는, 모델 훈련에 사용된 `mock_data.csv`가 명확한 특징 통계값을 지니도록 별도로 설계된 '가상 합성 데이터'이기 때문입니다.

## 5. 현재 한계 정리 (Limitations & Next Steps)

### ⚠️ 모델의 한계점
1. **데이터의 단순성과 과대적합 우려**
   - 매우 뚜렷한 규칙성을 가진 가상 데이터(Synthetic Data)를 사용하여 점수가 과하게 높습니다. 현실 세계의 노이즈(Noise) 데이터에 대해서는 일반화 성능을 100% 보장하기 어렵습니다.
2. **선형 모델 알고리즘의 한계**
   - 현재 적용된 로지스틱 회귀는 변수 간의 복잡하고 비선형적인 상호작용 패턴을 완벽히 매핑하는 데에는 태생적인 한계가 존재합니다.

### 🔧 향후 개선 방향
1. **비선형 모델(트리 기반 앙상블) 도입 모델 비교**
   - 다음 단계에서는 데이터 패턴을 단계적으로 여러 차례 분할할 수 있는 `Decision Tree`, 그리고 앙상블 기반의 `Random Forest`, `XGBoost` 등의 모델을 도입하여 각기 어떤 모델이 우수한지 비교 분석할 계획입니다.
2. **하이퍼파라미터 튜닝**
   - 교차 검증(Cross-Validation)을 기반으로 여러 알고리즘의 최적 파라미터를 찾는 과정을 거쳐 단단한 모델을 구축해야 합니다.
