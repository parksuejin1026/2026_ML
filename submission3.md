# 제출 3. 모델 개선 및 반복 수정 기록 보고서

## 1. 프로젝트 개요

본 보고서는 2차 제출에서 식별된 **클래스 불균형**과 **하이퍼파라미터 미최적화** 문제를 해결하기 위해 모델을 개선한 과정을 기록한 문서이다. `mock_data_3.csv`를 활용하여 전처리 로직을 고도화하고, SMOTE 기법과 Optuna 기반의 하이퍼파라미터 튜닝을 통해 **해지 후보(클래스 0) 탐지 성능(Recall)**을 극대화하는 데 초점을 맞추었다.

---

## 2. 모델 개선 과정 기록 (반복 수정 사항)

### 2-1. 데이터 전처리 변경 (Data Preprocessing)
| 변경 항목 | 구체적 내용 | 변경 이유 |
| --- | --- | --- |
| **SMOTE 적용** | 학습 데이터에 SMOTE를 적용하여 0과 1의 비율을 1:1로 조정 | 해지 후보(0) 데이터 부족으로 인한 학습 편향 해소 |
| **도메인 규칙 고도화** | `Value Gap`, `Churn Signal`, `Effective Cost` 산출 로직 최신화 | 사용자의 실질적 비용 부담과 이탈 의도를 더 정교하게 포착 |
| **Feature Scaling** | `StandardScaler`를 통한 모든 수치형 변수 정규화 | 모델(특히 CatBoost)의 학습 안정성 및 수렴 속도 향상 |

### 2-2. 하이퍼파라미터 조정 (Hyperparameter Tuning)
**Optuna**를 활용하여 CatBoost 모델의 최적 파라미터를 탐색하였다.
- **최적 파라미터**: `iterations: 432`, `learning_rate: 0.018`, `depth: 5`
- **조정 사유**: 
  - **`depth: 5`**: SMOTE를 통한 오버샘플링 데이터 학습 시 발생할 수 있는 노이즈 암기(과적합)를 방지하기 위해 트리의 깊이를 제한함. 이를 통해 모델의 일반화 성능을 확보함.
  - **`iterations: 432`**: 검증 데이터셋의 손실(Loss)이 최소화되는 조기 종료(Early Stopping) 지점을 반영하여, 불필요한 연산을 줄이고 학습 데이터에 대한 과도한 특화를 막음.
  - **`learning_rate: 0.018`**: 정밀한 가중치 업데이트를 위해 낮은 학습률을 채택하여, 전역 최적점(Global Optimum)에 안정적으로 수렴하도록 함.

### 2-3. 임계값(Threshold) 최적화
- **변경 사항**: 기본 분류 임계값인 0.5를 **0.4**로 하향 조정.
- **변경 이유**: 
  - **비즈니스 목적 부합**: 본 프로젝트의 핵심은 '이탈 위험군을 놓치지 않는 것(Recall)'임. 이탈할 사용자를 유지군으로 오판(FN)하는 리스크가 이탈하지 않을 사용자를 위험군으로 분류(FP)하는 리스크보다 훨씬 크기 때문에 임계값을 보수적으로 조정함.
  - **재현율 극대화**: 임계값을 0.4로 낮춤으로써 실제 해지 후보를 더 민감하게 포착하여 탐지 성능을 68%에서 75%로 향상시킴.

---

## 3. 모델 성능 비교 (결과 분석)

### 3-1. 개선 전/후 성능 비교표
> 평가 기준: 테스트셋 4,000건

| 지표 | 2차 제출 (Baseline - CatBoost) | 3차 제출 (Improved - CatBoost) | 변화량 |
| --- | :---: | :---: | :---: |
| **Recall (0)** | 0.6833 | **0.7500** | **+0.0667 (약 10%↑)** |
| Accuracy | 0.7350 | 0.6900 | -0.0450 (소폭 하락) |
| F1-score (0) | 0.6074 | 0.6300 | +0.0226 (상승) |
| ROC-AUC | 0.8184 | 0.7706 | -0.0478 (변동) |

### 3-2. 변화 원인 분석
1. **Recall(0)의 대폭 향상**: SMOTE를 통해 소수 클래스에 대한 학습량을 늘리고, 임계값을 0.4로 낮춤으로써 실제 해지 후보를 찾아내는 능력이 2차 제출 대비 약 10% 이상 향상되었다.
2. **Accuracy의 소폭 하락**: 이탈 탐지율을 높이기 위해 임계값을 낮춘 결과, 일부 유지 후보(1)를 해지 후보(0)로 오분류하는 경우가 늘어나며 전체 정확도는 다소 감소하였다. 그러나 '불필요한 구독 식별'이라는 비즈니스 목적에는 현재 모델이 더 부합한다.
3. **F1-score의 동반 상승**: Precision과 Recall의 균형 지표인 F1-score가 상승한 것은 단순히 탐지율만 높인 것이 아니라, 모델의 전반적인 분류 품질이 개선되었음을 의미한다.
